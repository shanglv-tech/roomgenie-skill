# RoomGenie 智能推荐系统设计文档

**日期**: 2026-04-15
**版本**: 1.0
**状态**: 设计中

---

## 1. 概述

### 1.1 项目目标

构建一个对话式的智能酒店推荐系统，提供：
- 精选酒店推荐
- 特价酒店筛选
- 专属度假顾问式的一对一需求挖掘

### 1.2 核心原则

- **对话优先**: 以自然语言对话为主要交互方式
- **无压力追问**: 最多追问3次，提供选项，随时可跳过
- **本地优先**: 用户画像存储在本地 `~/.roomgenie/`
- **渐进式**: 第一阶段聚焦推荐能力，下单功能后续迭代

## 3. 对话体验设计

### 3.1 无压力追问原则

1. **选项优先，开放式为辅**
   - ✅ 好: "预算大概在哪个范围？A) ¥300以下 B) ¥300-600 C) ¥600-1000 D) 都可以看看"
   - ❌ 不好: "你预算多少？"

2. **最多连续追问3次**
   - 超过3次就直接基于已有信息推荐

3. **"随时跳过"暗示**
   - 每次追问都暗示用户可以说"先这样吧"、"直接推荐"来跳过

4. **模糊容忍度高**
   - 用户说"随便"、"都行"、"看情况"就接受，不再追问

### 3.2 需求挖掘流程

```
开始
  ↓
已有明确信息？（目的地+日期+预算）
  ├─ 是 → 直接推荐
  └─ 否 → 温和追问（最多3轮）
              ↓
         收集到足够信息？
              ├─ 是 → 推荐
              └─ 否 → 基于部分信息推荐，告知"如需更精准可补充信息"
```

### 3.3 对话示例

```
用户: 我周末想出去住两天

Skill: 好呀！几个小问题帮你找最合适的～
      1) 有想去的城市吗？
      2) 还是想让我推荐几个好去处？
      (可以直接说"先这样吧"跳过问题)

用户: 还没定，你推荐吧

Skill: 收到！预算大概在哪个范围呢？
      A) ¥300以下  B) ¥300-600  C) ¥600-1000  D) 都行

用户: 都行，你看着办

Skill: 好嘞，这就为你推荐几个不错的选择！
      (基于热门目的地+中等预算推荐)
```

### 4.3 画像更新策略

- 每次搜索后记录到 `recentSearches`
- 用户明确表达偏好时更新 `preferences`
- 不做过度推断，只在用户明确反馈时更新

## 5. 推荐逻辑设计

### 5.1 推荐类型

| 类型 | 触发关键词 | 逻辑 |
|------|-----------|------|
| 精选推荐 | "精选"、"好的"、"品质"、"不错的" | 高评分、好评数优先，结合用户偏好 |
| 特价推荐 | "特价"、"便宜"、"性价比"、"优惠" | 价格从低到高，筛选评分≥4.0的 |
| 综合推荐 | 其他情况 | 平衡评分、价格、用户偏好 |

### 5.2 结果展示

1. 推荐3-5个选项
2. 每个选项展示：名称、评分、价格、地址、核心设施
3. 展示酒店图片（如果有）
4. 展示预订链接：`[Book now]({bookingUrl})`
5. 提示用户："如需预订可点击上方链接"

## 6. Skill 激活方式

### 6.1 自然语言触发（主要方式）

通过 SKILL.md 中定义的 patterns 自动匹配：

- "((search|find|recommend|compare|book).*(hotel|stay|accommodation|resort|inn|hostel|homestay))"
- "((酒店|住宿|民宿|度假村).*(搜索|查找|推荐|比较|预订|查询|价格))"
- 等等...

### 6.2 Slash Command（辅助方式）

`/roomgenie` - 主入口，对话式推荐
